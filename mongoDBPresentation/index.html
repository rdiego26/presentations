Alguns conceitos...

  • Nunca, mas nunca está relacionado ao ódio ao modelo SQL, pelo contrário, podem até trabalhar em conjunto;
  • O termo NoSQL e Não Relacional também é bastante utilizado;
  • Orientado a documento e não a registro;

Nomenclatura

Relacional vs NoSQL

  • Tabela -> Coleção
  • Registro -> Documento
  • Índice -> Índice
  • Join -> Documento embarcado
  • Foreign Key -> Referência

Recursos Like a Boss

  • Full Text Search
  • Índices Espaciais (geográficos) - GeoJSON
  • Sharding - Clusterização
  • Replica Set - Alta Disponibilidade

Considere uso do MongoDB, quando..

  • Está usando muito cache em sua aplicação;
  • Precisa de processamento em tempo real;
  • Gosta de desenvolvimento ágil;
  • Sua aplicação é "beta perpétua";
  • Usa muito "join" na sua aplicação relacional;

CRUD Operations

O console pode ser uma ótima ferramenta de aprendizado. :)

                            
                                ramos@e9:~$ mongo
                                MongoDB shell version: 2.0.6
                                connecting to: test
                                >

Insert

                            
                                > db.workshop.insert({
                                    company: 'E-novasys',
                                    website: 'http://enovasys.net.br/'
                                })
                            
                        
                            
                                > db.workshop.insert({
                                    company:'otherCompany',
                                    technologies: ['NoSQL', 'Oracle']
                                })

Consultando um registro

                            
                                > db.workshop.findOne()
                                {
                                "_id" : ObjectId("52aa872c7eea1e900f218d5c"),
                                "company" : "E-novasys",
                                "website" : "http://enovasys.net.br/"
                                }

Consultando um registro - filtro

                            
                                > db.workshop.find({company:'E-novasys'}).pretty()
                                {
                                "_id" : ObjectId("52aa872c7eea1e900f218d5c"),
                                "company" : "E-novasys",
                                "website" : "http://enovasys.net.br/"
                                }
pretty() é para formatar melhor a saída

Checando existência de campo - filtro

                            
                                > db.workshop.find( {technologies: {$exists: true} } )
                                { "_id" : ObjectId("52aa8cf4c7b09b7b3ff24969"), "company" : "otherCompany", "technologies" : [ "NoSQL", "Oracle", "Linux" ] }

Quantidade de registros

                            
                                > db.workshop.count()
                                2

Atualizando registro

                            
                                > db.workshop.update({ company:'E-novasys' }, {$set: { technologies:['Oracle', 'Linux'] } } )
Caso o campo não exista, o mongoDB irá inserir no documento

Removendo registro

                            
                                > db.workshop.remove({company:'otherCompany'})

Comandos úteis
